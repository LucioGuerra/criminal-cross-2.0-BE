    - If document is null, return null
    - Otherwise, convert to domain using the existing `toDomain(document)` private method

    **3. Create DeleteActivityScheduleUseCase** (`@ApplicationScoped`, `execute(Long id)` method):
    - Inject `ActivityScheduleRepository`
    - Validate: `id` must not be null (throw `BadRequestException("Schedule ID is required")`)
    - Load schedule: `activityScheduleRepository.findById(id)` — if null, throw `EntityNotFoundException("Activity schedule not found")`
    - Set `active = false` on the loaded schedule (soft delete)
    - Save: `activityScheduleRepository.save(schedule)` — the existing `save()` already handles updates when `id` is set
    - Return the updated schedule (with active=false)

    **4. Create UpdateActivityScheduleUseCase** (`@ApplicationScoped`, `execute(Long id, ActivitySchedule updatedData)` method):
    - Inject `ActivityScheduleRepository`
    - Validate: `id` must not be null (throw `BadRequestException("Schedule ID is required")`)
    - Validate: `updatedData` must not be null (throw `BadRequestException("Schedule data is required")`)
    - Load existing: `activityScheduleRepository.findById(id)` — if null, throw `EntityNotFoundException("Activity schedule not found")`
    - Update mutable fields on the existing schedule from `updatedData` (only non-null fields from the update request should be applied):
      - `dayOfWeek`, `weekDays`, `startTime`, `durationMinutes`, `active`, `schedulerType`, `templateType`, `activeFrom`, `activeUntil`, `scheduledDate`
    - Do NOT allow updating: `id`, `organizationId`, `headquartersId`, `activityId` (these are identity/ownership fields)
    - Set the schedule's ID to the original `id` (ensure it's preserved for the save method)
    - Save: `activityScheduleRepository.save(existing)` — returns the updated schedule
    - Return the saved result

    Follow the exact same pattern as `CreateActivityScheduleUseCase` for CDI annotation, field injection, and exception handling. Use plain POJO convention (no Lombok). Import `BadRequestException` and `EntityNotFoundException` from `org.athlium.shared.exception`.

    ```java
    @DELETE
    @Path("/{id}")
    public Response delete(@PathParam("id") Long id) {
        try {
            var deleted = deleteActivityScheduleUseCase.execute(id);
            return Response.ok(ApiResponse.success("Schedule deleted", mapper.toResponse(deleted))).build();
        } catch (EntityNotFoundException e) {
            return Response.status(Response.Status.NOT_FOUND)
                .entity(ApiResponse.error(e.getMessage())).build();
        } catch (BadRequestException e) {
            return Response.status(Response.Status.BAD_REQUEST)
                .entity(ApiResponse.error(e.getMessage())).build();
        } catch (Exception e) {
            return Response.status(Response.Status.INTERNAL_SERVER_ERROR)
                .entity(ApiResponse.error("Unexpected error")).build();
        }
    }
    ```

    **4. Add PUT endpoint:**
    ```java
    @PUT
    @Path("/{id}")
    public Response update(@PathParam("id") Long id, ActivityScheduleRequest request) {
        try {
            var updated = updateActivityScheduleUseCase.execute(id, mapper.toDomain(request));
            return Response.ok(ApiResponse.success("Schedule updated", mapper.toResponse(updated))).build();
        } catch (EntityNotFoundException e) {
            return Response.status(Response.Status.NOT_FOUND)
                .entity(ApiResponse.error(e.getMessage())).build();
        } catch (BadRequestException e) {
            return Response.status(Response.Status.BAD_REQUEST)
    ```
